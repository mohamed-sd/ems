# 🔐 دليل نظام الصلاحيات المتقدم - Role Permissions System

## ⭐ نظرة عامة

تم إضافة نظام صلاحيات متقدم يسمح بالتحكم الدقيق في صلاحيات كل دور على كل شاشة (module) في النظام.

---

## 📊 المكونات الرئيسية

### 1️⃣ جدول `role_permissions` 

**الموقع:** قاعدة البيانات `equipation_manage`

```sql
CREATE TABLE role_permissions (
  id         INT PRIMARY KEY AUTO_INCREMENT,
  role_id    INT NOT NULL,
  module_id  INT NOT NULL,
  can_view   BOOLEAN DEFAULT FALSE,    -- 👁️ يمكن العرض
  can_add    BOOLEAN DEFAULT FALSE,    -- ➕ يمكن الإضافة
  can_edit   BOOLEAN DEFAULT FALSE,    -- ✏️ يمكن التعديل
  can_delete BOOLEAN DEFAULT FALSE,    -- 🗑️ يمكن الحذف
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  
  FOREIGN KEY (role_id) REFERENCES roles(id),
  FOREIGN KEY (module_id) REFERENCES modules(id),
  UNIQUE(role_id, module_id)
);
```

#### حقول الصلاحيات:
- **👁️ can_view**: السماح برؤية الشاشة والبيانات
- **➕ can_add**: السماح بإضافة سجلات جديدة
- **✏️ can_edit**: السماح بتعديل السجلات الموجودة
- **🗑️ can_delete**: السماح بحذف السجلات

### 2. صفحة إدارة الصلاحيات المتقدمة
**المسار:** `/Settings/role_permissions.php`

هذه هي الصفحة الرئيسية لإدارة الصلاحيات، توفر:

#### 🎯 ثلاث طرق عرض مختلفة:

##### أ) عرض الشبكة (Grid View)
- اختر دوراً من قائمة منسدلة
- شبكة بطاقات لكل شاشة
- كل بطاقة تحتوي على 4 checkboxes للصلاحيات
- زر لحفظ الصلاحيات لكل شاشة
- أزرار سريعة:
  - ✅ منح الكل - منح جميع الصلاحيات للدور
  - 🚫 سحب الكل - إزالة جميع الصلاحيات

##### ب) عرض الجدول (Table View)
- جدول يعرض جميع الصلاحيات المحفوظة
- عمود لكل صلاحية مع رموز مرئية
- خيار حذف الصلاحية
- سهل البحث والتصفية

##### ج) الإجراءات السريعة (Quick Actions)
- منح جميع الصلاحيات لدور معين
- سحب جميع الصلاحيات من دور معين
- جدول مرجعي يشرح الصلاحيات

## 📋 طريقة الاستخدام

### الخطوة 1️⃣: إضافة الدور والشاشة (Module)

أولاً، تأكد من وجود:
- ✅ الأدوار (Roles) في جدول `roles` - ذهب إلى `Settings/roles.php`
- ✅ الشاشات (Modules) في جدول `modules` - ذهب إلى `Settings/modules.php`

### الخطوة 2️⃣: إنشاء الجدول

1. ذهب إلى `/setup_role_permissions.php`
2. انقر على **"إنشاء الجدول الآن"**
3. تأكد من ظهور رسالة النجاح ✅

### الخطوة 3️⃣: تعيين الصلاحيات

**الطريقة الأولى - العرض الشبكة:**
1. ذهب إلى `Settings/role_permissions.php`
2. انقر على تبويب "عرض الشبكة"
3. اختر الدور من القائمة
4. حدد الصلاحيات لكل شاشة
5. انقر "حفظ"

**الطريقة الثانية - الإجراءات السريعة:**
1. انقر على تبويب "الإجراءات السريعة"
2. اختر الدور
3. انقر "منح الكل" أو "سحب الكل"

## 🛠️ العمليات المتاحة

### 1. حفظ صلاحية جديدة
```php
POST /Settings/role_permissions.php
{
    "action": "save_permissions",
    "role_id": 1,
    "module_id": 5,
    "can_view": 1,
    "can_add": 1,
    "can_edit": 1,
    "can_delete": 0
}
```

**النتيجة:**
- إذا لم تكن هناك صلاحية سابقة → **إدراج جديد**
- إذا كانت هناك صلاحية سابقة → **تحديث**

### 2. حذف صلاحية
```php
POST /Settings/role_permissions.php
{
    "action": "delete_permission",
    "id": 42
}
```

### 3. منح جميع الصلاحيات
```php
POST /Settings/role_permissions.php
{
    "action": "grant_all",
    "role_id": 1
}
```

**الفعل:**
- حذف جميع الصلاحيات السابقة للدور
- إضافة صلاحية جديدة لكل شاشة (4/4 مفعلة)

### 4. سحب جميع الصلاحيات
```php
POST /Settings/role_permissions.php
{
    "action": "revoke_all",
    "role_id": 1
}
```

**الفعل:**
- حذف جميع الصلاحيات للدور

## 🔐 الحماية الأمنية

✅ **التحقق من الصلاحيات:**
```php
// التحقق من أن المستخدم له إذن بعرض شاشة معينة
$stmt = $conn->prepare(
    "SELECT can_view FROM role_permissions 
     WHERE role_id = ? AND module_id = ?"
);
$stmt->bind_param("ii", $_SESSION['user']['role'], $module_id);
$stmt->execute();
$result = $stmt->get_result()->fetch_assoc();

if (!$result || !$result['can_view']) {
    die("❌ لا توجد صلاحيات للوصول إلى هذه الشاشة");
}
```

✅ **CSRF Protection**: تتم معالجة جميع الطلبات عبر POST

✅ **Input Validation**: التحقق من جميع المدخلات وتنظيفها

✅ **Role-Based Access**: فقط المسؤول يمكنه إدارة الصلاحيات

## 🐛 استكشاف الأخطاء

### الخطأ: "الجدول غير موجود"
**الحل:** ذهب إلى `/setup_role_permissions.php` وانقر "إنشاء الجدول الآن"

### الخطأ: "Foreign Key Constraint Failed"
**السبب:** الدور أو الشاشة غير موجود
**الحل:** تأكد من وجود الدور والشاشة أولاً

### الخطأ: "لا توجد صلاحيات"
**التحقق من:**
1. تأكد من أن الصلاحية تم حفظها ✅
2. تحقق من معرف الدور والشاشة 🔍
3. امسح الـ Cache إن وجد 🗑️
