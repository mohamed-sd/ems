# برومبت: شاشة شجرة العميل (عميل ← مشاريع ← موردون ← معدّات ← مشغّلون) — Claude Code

> انسخ كل ما تحت الخط والصقه في Claude Code داخل مجلد المشروع `c:/xampp/htdocs/ems`.

---

## الهدف

أنشئ شاشة جديدة **«شجرة العميل»** تعرض هرماً قابلاً للطيّ بخمسة مستويات:
**العميل (الجذر) ← مشاريعه ← موردو كل مشروع ← معدّات كل مورّد ← مشغّلو كل معدّة.**

شاشة **عرض فقط** (لا كتابة). نظام EMS: PHP خام + MySQLi، عربي RTL، متعدّد الشركات، XAMPP.

## أعِد استخدام منطق موجود (لا تبدأ من الصفر)

**اقرأ `movement/map_page.php` بالكامل** — فهو يجمّع جوهر البيانات (مورّد ← معدّات ← مشغّلون) لمشروع واحد في المصفوفة `$suppliers_data`:
- `operations` (المعدّات المشغّلة) JOIN `equipments` JOIN `equipments_types` LEFT JOIN `suppliers`، مجمّعة **حسب المورّد**.
- لكل معدّة: المشغّلون النشطون من `equipment_drivers` JOIN `drivers`.
- (اختياري) ساعات التايم شيت لكل معدّة.

**الجديد هنا:** غلّف هذا المنطق بمستويين أعلى:
1. **العميل (الجذر):** من جدول `clients` (تحقّق من الاسم الفعلي للجدول/الأعمدة بنفسك) مع عزل الشركة. الشاشة إمّا تعرض **عميلاً محدّداً** (`?client_id=`) أو **كل عملاء الشركة** كجذور متعدّدة قابلة للطيّ (الافتراضي).
2. **المشاريع تحت كل عميل:** من جدول `projects` حيث `project.client_id = <client>` (تحقّق من اسم عمود الربط) مع عزل الشركة. ثم **لكل مشروع** طبّق نفس تجميع `map_page.php` (الموردون ← المعدّات ← المشغّلون).

**استخرج استعلامات التجميع وقواعد العزل/الصلاحيات من `map_page.php`** وابنِ فوقها — لا تكرّر منطقاً مختلفاً. وللكفاءة اجلب البيانات بعدد محدود من الاستعلامات (مثلاً: كل العملاء، ثم كل المشاريع، ثم استعلام عمليات واحد مفلتر بمشاريع الشركة، ثم تجميعها في PHP) لا استعلاماً منفصلاً لكل عقدة.

اقرأ أيضاً: `config.php`, `includes/permissions_helper.php` (`check_page_permissions`, `db_table_has_column`), `includes/page_header.php`, `inheader.php`/`insidebar.php`, `assets/css/design-tokens.css`، وشاشة العملاء/المشاريع الموجودة لمعرفة أسماء الجداول والأعمدة الحقيقية.

## الشاشة: `movement/client_tree.php`

### البنية (شجرة هرمية قابلة للطيّ — 5 مستويات)
- **المستوى 0 — العميل (الجذر):** اسم العميل + كوده، مع إجماليات مجمّعة (عدد المشاريع · إجمالي الموردين · إجمالي المعدّات · عاملة · متوقفة · إجمالي المشغّلين · **إجمالي الساعات المنفّذة**).
- **المستوى 1 — المشاريع:** كل مشروع عقدة قابلة للطيّ تحت العميل: الاسم + الكود + الموقع + **شارات** (موردون · معدّات · عاملة · متوقفة · مشغّلون) + **إجمالي ساعات المشروع** (مجموع ساعات كل مورّديه).
- **المستوى 2 — المورّدون:** تحت كل مشروع، كل مورّد عقدة: اسمه + شارات (عدد معدّاته · عاملة · متوقفة · عدد مشغّليه) + **ساعات المورّد** (مجموع ساعات معدّاته) + **نسبته من ساعات المشروع %** (لإظهار توزيع الحصص بصرياً، مثل شريط نسبة).
- **المستوى 3 — المعدّات:** تحت كل مورّد، كل معدّة عقدة: الكود + النوع + **نقطة حالة** (عاملة/متوقفة) + عدّاد مشغّليها + **ساعاتها** (اليوم/الإجمالي).
- **المستوى 4 — المشغّلون:** تحت كل معدّة، كل مشغّل ورقة: الاسم + الكود + الهاتف + الوردية (☀️/🌙/🔄). إن لا مشغّل: «لا مشغّل».

### الساعات وتوزيع الحصص (مهم)
الهدف معرفة **كم ساعة يملك كل مورّد** ضمن المشروع لرؤية توزيع الحصص:
- **مصدر الساعات الفعلية:** التايم شيت — `timesheet.operator = operations.id` (نفس ربط `map_page.php` لحساب ساعات كل معدّة). اجمعها تصاعدياً: معدّة → مورّد → مشروع → عميل.
- **حدّد نطاق الساعات بوضوح في الواجهة:** أتِح اختيار الفترة (اليوم / هذا الشهر / الإجمالي التراكمي / مدى تاريخي) عبر فلتر بسيط في الرأس، وطبّقه على كل المستويات. الافتراضي: الشهر الحالي.
- **الحصة التعاقدية (إن توفّرت):** إن وُجدت ساعات مخطّطة/مخصّصة في العقود أو في `operations` (مثل `shift_hours`/ساعات العقد لكل معدّة — تحقّق من الأعمدة الفعلية)، اعرض لكل مورّد ومشروع: **المخصّص مقابل المنفّذ** مع **نسبة الاستهلاك %** وتلوين تحذيري عند التجاوز (متّسق مع منطق تقرير الحصص `database/quota_overallocation_report.sql` إن وُجد). إن لم تتوفّر بيانات حصص موثوقة، اكتفِ بعرض الساعات المنفّذة ونسبة كل مورّد من إجمالي المشروع، ولا تختلق أرقام حصص.
- **توزيع الحصص بصرياً:** عند عقدة المشروع اعرض تفكيكاً سريعاً (شريط/نِسَب) لمساهمة كل مورّد في إجمالي ساعات المشروع.

### التفاعل (JavaScript خام، بلا مكتبات ثقيلة)
- **طيّ/فتح** كل عقدة بالنقر (سهم/أيقونة دوّارة). افتراضي: العملاء والمشاريع مفتوحة، الموردون والمعدّات مطويّة (لتفادي ازدحام 5 مستويات).
- أزرار **«فتح الكل» / «طيّ الكل»**.
- **بحث/تصفية فوري** عبر كل المستويات (يُبرز العقد المطابقة بالاسم/الكود ويُبقي مساراتها مفتوحة حتى الجذر).
- (اختياري) فلاتر سريعة: المعدّات العاملة فقط / المتوقفة فقط.
- خطوط ربط بصرية (indentation + خطوط) توضّح الهرمية، وألوان دلالية متّسقة (أخضر=عاملة، أحمر=متوقفة).

### الروابط
- اسم المشروع → `map_page.php?project_id=<id>` و/أو `movement_operations.php?project_id=<id>`.
- كود المعدّة → `../Equipments/equipment_profile.php?id=<eq_id>`.
- اسم المشغّل → `../Drivers/driver_profile.php?id=<driver_id>`.
- زر رجوع للوحة التحكم، و(اختياري) زر تصدير/طباعة للشجرة.

## القواعد الإلزامية (على نمط النظام)
- حارس الجلسة → `config.php` → `check_page_permissions($conn, 'movement/client_tree.php')` (صلاحية عرض على الأقل) → عزل الشركة (`company_id`) عبر `db_table_has_column` على العملاء والمشاريع والعمليات. الأدوار المقيّدة (مثل مدير الموقع المحصور بمشروع) ترى فقط نطاقها — اعرض عميل/مشروع ذلك المستخدم فقط.
- ترويسة موحّدة عبر `page_header.php`، RTL، رموز الهوية (`design-tokens.css`).
- كل المخرجات `htmlspecialchars`/`e()`، كل الاستعلامات **Prepared Statements** أو نفس أسلوب `map_page.php` الآمن.
- لا كتابة في قاعدة البيانات إطلاقاً (عرض فقط).
- حالات فارغة أنيقة على كل مستوى: «لا مشاريع لهذا العميل»، «لا موردون/معدّات في هذا المشروع».

## التنقّل والصلاحيات
- أضِف الشاشة إلى جدول `modules` (تابعة لدور مناسب — مثل مدير الحركة/التشغيل أو المشروعات/العملاء) بكود `movement/client_tree.php` وأيقونة شجرة (`fa fa-sitemap`)، لتظهر في القائمة الديناميكية وتُفحص صلاحيتها.

## التحقق (إلزامي قبل الإنهاء)
1. `php -l movement/client_tree.php` — بلا أخطاء.
2. على XAMPP بعميل له عدّة مشاريع، كل مشروع فيه موردون ومعدّات ومشغّلون: تأكّد من بناء الشجرة الصحيحة (العميل ← مشاريعه ← مورّدوه ← معدّاته ← مشغّلوه)، عمل الطيّ/الفتح والبحث عبر المستويات، وتطابق إجماليات كل مشروع مع `map_page.php` لنفس المشروع.
   - **تحقّق الساعات:** ساعات المعدّات تطابق `map_page.php`، ومجموع ساعات معدّات المورّد = ساعات المورّد، ومجموع ساعات الموردين = إجمالي ساعات المشروع، ونِسَب توزيع المورّدين تجمع 100%. تحقّق من اتساق الأرقام عند تغيير فلتر الفترة.
3. تأكّد من عزل الشركة والصلاحيات (عميل/شركة أخرى لا يظهر، والدور المقيّد يرى نطاقه فقط)، وأن الشاشة لم تكسر شيئاً (إضافية بحتة).

## المخرجات المطلوبة
- `movement/client_tree.php` (الشاشة) + أي CSS/JS لازم (يفضّل ملف `assets/css/client-tree.css` معزول، أو ضمن الصفحة).
- سكربت SQL لإضافة الوحدة إلى `modules`.
- ملخّص: مصدر البيانات (المُعاد استخدامه من `map_page.php` + جداول `clients`/`projects`)، بنية الشجرة الخماسية، وتأكيد عدم الكسر.

**ابدأ بقراءة `map_page.php` لاستخراج منطق التجميع والعزل، وتحقّق من أسماء جداول/أعمدة `clients` و`projects` فعلياً، ثم ابنِ الشجرة الخماسية القابلة للطيّ بأقل عدد استعلامات، ثم تحقّق.**
