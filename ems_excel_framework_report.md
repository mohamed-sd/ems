# تقرير تصميم نظام Excel الموحد (Unified Excel Framework)

> **تاريخ:** 4 يونيو 2026
> **النطاق:** طبقة موحّدة للاستيراد والتصدير من/إلى Excel على مستوى النظام بالكامل.
> **المبدأ الحاكم:** إضافة طبقة جديدة دون كسر أي وظيفة قائمة، وجعلها المرجع الرسمي الوحيد لاحقاً.

---

## 1) الوضع الحالي

يوجد تعامل مع Excel في **5 وحدات** عبر ملفات منفصلة شبه متطابقة:

| الوحدة | استيراد | تصدير | نموذج Excel | نموذج CSV |
|--------|:---:|:---:|:---:|:---:|
| العملاء (Clients) | ✅ | ✅ | ✅ | ✅ |
| السائقون (Drivers) | ✅ | ❌ | ✅ | ✅ |
| المعدات (Equipments) | ✅ | ❌ | ✅ | ✅ |
| الموردون (Suppliers) | ✅ | ❌ | ✅ | ✅ |
| المشاريع (Projects) | ✅ (+عملاء) | ❌ | ✅ | ✅ |
| التقارير (emsreports) | — | ✅ (نظام منفصل) | — | — |

كل ملف يبلغ ~150–210 سطراً، ومجموع هذه الملفات يكرّر نفس المنطق عشرات المرات.

## 2) المشاكل الحالية

1. **تكرار ضخم (DRY violation):** كل وحدة تعيد كتابة: التحقق من الجلسة، نطاق الشركة، البحث عن صلاحية الوحدة، تنسيق PhpSpreadsheet، معالجة CSV/BOM/الترميز، ترويسات بث الملف.
2. **لا يوجد تصدير إلا للعملاء:** بقية الجداول بلا تصدير رغم الحاجة.
3. **الاستيراد بدائي:** رفع → إدراج مباشر. **لا توجد شاشة معاينة (Preview)**، ولا تقرير أخطاء منظّم، ولا فصل بين «صحيح/خاطئ» قبل الحفظ.
4. **ثغرة أمنية — لا حماية CSRF** في نقاط الاستيراد (`import_*_excel.php` لا تُطابق حارس AJAX في `config.php`، ولا تتحقق من توكن CSRF).
5. **بناء SQL بالتسلسل النصّي** مع هروب يدوي وفحص تكرار صف-بصف (N+1).
6. **هوية بصرية غير متّسقة:** ملفات Excel الحالية تستخدم اللون الكحلي (#1e3a5f) بينما هوية النظام **ذهبية/سوداء** (`--brand-amber #F2AA2A`، `--brand-black #0F1115`).
7. **حلّ الصلاحيات هشّ:** بحث `LIKE '%clients.php%'` في جدول `modules`.
8. **واجهة الاستيراد مكتوبة يدوياً** داخل كل صفحة (Modal + JS) — غير قابلة لإعادة الاستخدام.

## 3) أماكن التكرار (ملخص)

`download_*_template.php` ×6، `download_*_template_csv.php` ×5، `import_*_excel.php` ×6، `export_*_excel.php` ×1 — جميعها أنماط متطابقة قابلة للدمج في خدمة واحدة + سجلّ تعريفات (Registry).

## 4) المعمارية المقترحة

طبقة موحّدة تحت مساحة الأسماء `App\Services\Excel` (مستفيدة من مُحمّل PSR-4 الموجود في `app/bootstrap.php`)، يقودها **سجلّ تعريفات مركزي** ونقطة دخول واحدة.

```
                       ┌──────────────────────────┐
   المتصفح  ───────►   │   excel.php (Front Ctrl)  │   نقطة الدخول الوحيدة
  (ems-excel.js)       └────────────┬─────────────┘
                                    │
                       ┌────────────▼─────────────┐
                       │     ExcelService (Facade) │  حُرّاس: Auth + CSRF + Permission + Scope
                       └────────────┬─────────────┘
        ┌──────────────┬────────────┼─────────────┬──────────────┐
        ▼              ▼            ▼              ▼              ▼
   TemplateBuilder  Exporter     FileReader     Validator      Importer
        │              │            │              │              │
        └──────────────┴──────┬─────┴──────────────┴──────────────┘
                              ▼
                  Styler (هوية ذهبية/سوداء + RTL + تجميد + تنسيق)
                              ▲
                  ExcelRegistry ◄── EntityDefinition + Column  (مصدر الحقيقة الوحيد)
```

**المكوّنات:**

| المكوّن | المسؤولية |
|--------|-----------|
| `EntityDefinition` + `Column` | وصف الكيان: الجدول، الأعمدة (نوع/مطلوب/فريد/قيم/مفتاح أجنبي/مثال)، نطاق الشركة. |
| `ExcelRegistry` | تسجيل الكيانات: مصدر الحقيقة الوحيد. إضافة كيان = سطور قليلة. |
| `Styler` | تنسيق احترافي موحّد بالهوية البصرية + RTL + تجميد الصفوف + تنسيق التواريخ/الأرقام. |
| `Exporter` | تصدير بيانات (كل/مفلتر/محدد) إلى Excel منسّق. |
| `TemplateBuilder` | بناء نموذج جاهز (ورقة بيانات + أمثلة + ورقة تعليمات). |
| `FileReader` | قراءة آمنة xlsx/xls/csv (UTF-8/BOM/Windows-1256) + فحوص أمنية. |
| `Validator` | تحقق صف-بصف (مطلوب/نوع/قيم/بريد/هاتف/تاريخ/تكرار داخلي وقاعدي/مفاتيح أجنبية). |
| `Importer` | معاينة (Preview) ثم تنفيذ (Commit) بـ Prepared Statements داخل Transaction. |
| `ExcelService` | الواجهة الموحّدة + كل الحُرّاس الأمنية. |
| `excel.php` | المتحكّم الأمامي الوحيد (`?entity=...&action=template|export|import_preview|import_commit`). |
| `ems-excel.js` / `ems-excel.css` | مكوّن واجهة موحّد: أزرار + معالج استيراد متعدد الخطوات + شريط تقدّم. |
| `includes/excel_ui.php` | مساعد PHP: يضيف الأزرار + نافذة المعالج لأي صفحة بسطرين. |

**تجربة الاستيراد (6 خطوات كما طُلب):** تحميل نموذج ← رفع ← **معاينة** (إجمالي/صحيح/خاطئ/تحذيرات) ← تحقق ← **تقرير أخطاء** (صف/عمود/الخطأ/كيفية الإصلاح، قابل للتنزيل) ← تنفيذ (الصحيح فقط).

**الأمان:** Auth + CSRF + صلاحية الوحدة + نطاق الشركة + فحص الامتداد/MIME/الحجم + حد الصفوف + Transaction.

**الأداء:** قراءة بوضع البيانات فقط (`setReadDataOnly`)، إدراج عبر Prepared Statement مُعاد الاستخدام داخل Transaction، تخزين مؤقت للمعاينة في `storage/` لتفادي رفع مزدوج، حدود حجم/صفوف.

## 5) خطة التنفيذ

1. **النواة:** `Column`, `EntityDefinition`, `ExcelRegistry`, `Styler`.
2. **العمليات:** `Exporter`, `TemplateBuilder`, `FileReader`, `Validator`, `Importer`, `ExcelService`.
3. **نقطة الدخول:** `excel.php`.
4. **الواجهة:** `ems-excel.js`, `ems-excel.css`, `includes/excel_ui.php`.
5. **التسجيل:** تعريف clients/drivers/suppliers/equipments/projects في السجلّ.
6. **الدمج المرجعي:** تطبيق كامل على شاشة العملاء (دون حذف الملفات القديمة — غير كاسر).
7. **التوثيق:** دليل المطوّر + قائمة طرح لبقية الشاشات.

**ملاحظة نطاق صريحة:** يُسلَّم في هذه المرحلة *إطار العمل الكامل وقابل للتشغيل* + *دمج مرجعي مُثبت على شاشة العملاء* + *تعريفات جاهزة لبقية الكيانات الرئيسية*. تحويل بقية الشاشات لاستخدام الإطار يتم بتعديل سطور قليلة فيها وفق دليل المطوّر، دون أي تغيير في منطق العمل أو قاعدة البيانات.
