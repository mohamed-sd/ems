# برومبت: جداول أبناء كرت المعدة (وثائق · حماية · مكوّنات · تاريخ) — Claude Code

> انسخ كل ما تحت الخط والصقه في Claude Code داخل مجلد المشروع `c:/xampp/htdocs/ems`.
> **المرحلة 0 — الخطوة 4** من مشروع «الأسطول والجاهزية» (راجع `ROADMAP_fleet_readiness_on_php.md`). يأتي بعد توسعة `equipments` إلى كرت المعدة (`PROMPT_fleet_equipment_card.md`).

---

## السياق والنطاق

نظام EMS (PHP خام + MySQLi، عربي RTL، متعدّد الشركات، XAMPP). جدول `equipments` أصبح **كرت المعدة**. نبني الآن **جداول الأبناء (One2many)** المرتبطة بالكرت:

1. **الوثائق الرسمية** (§9.4-أ) — رخص/تأمينات/شهادات.
2. **تجهيزات الحماية** (§9.4-ب).
3. **المكوّنات الكبرى** (§9.4-ج).
4. **سجل تاريخ المعدة** (§9.6) — إدراج فقط.

> النطاق: إنشاء الجداول الأربعة + إدارتها (سطور قابلة للتكرار) في صفحة تفاصيل المعدة/ملفها + التنبيهات الأساسية. **بلا** حساب إهلاك ولا `engine_hours` تراكمي ولا تكامل S08 (الحقول المحسوبة من S08 تبقى فارغة/«لاحقاً»). كلها مرتبطة بـ `equipment_id` (= `equipments.id`) وبعزل الشركة.

## اقرأ أولاً

- `Equipments/equipments.php`, `equipment_profile.php`, `get_equipment_details.php` — نمط الكرت وأين تُضاف أقسام السطور.
- `Drivers/drivercontracts.php` — **نمط جداول السطور القابلة للتكرار** (إضافة/حذف سطر + رفع ملف) المعتمد في النظام؛ احتذِ به.
- `includes/permissions_helper.php` (`db_table_has_column`, `check_page_permissions`) · `config.php` · `database/equipation_manage.sql`.

## القواعد العامة لكل الجداول

- كل جدول: `id` PK AI، `company_id` (عزل)، `equipment_id` (FK → `equipments.id`)، حقول الجدول، `created_by`, `created_at`, `updated_at`، و(حيث يلزم) `is_deleted` للحذف الناعم.
- رفع الملفات إلى مجلد محمي (مثل `storage/fleet/` مع `.htaccess` كما في إطار Excel) وتخزين المسار فقط.
- الحقول **المحسوبة من S08/العدّاد** (مثل `replace_date`, `component_hours`, `replace_count`, حالة الوثيقة المحسوبة) تُترك فارغة الآن وتُعرض «لاحقاً/—» — لا تبنِ مصدرها بعد.
- كل الكتابة Prepared Statements + CSRF، كل المخرجات `htmlspecialchars`/`e()`.

## 5) الواجهة (في ملف المعدة / تفاصيل الكرت)

- في `equipment_profile.php` (وملحقاً في تفاصيل المعدة): أضِف **أربعة أقسام** (تبويبات أو بطاقات قابلة للطيّ): الوثائق · الحماية · المكوّنات · سجل التاريخ.
- كل قسم: جدول بسطور موجودة + نموذج/مودال «إضافة سطر» (وحذف ناعم للأبناء عدا التاريخ) على نمط `drivercontracts.php`.
- شارات تنبيه: الوثائق المنتهية/المقاربة (أحمر/برتقالي)، الحماية «يحتاج تجديداً».
- **سجل التاريخ**: عرض كخط زمني للقراءة + زر «إضافة حدث يدوي» فقط (لا تعديل/حذف).

## التحقق (إلزامي قبل الإنهاء)

1. `php -l` على كل ملف معدّل/جديد — بلا أخطاء.
2. شغّل SQL على `equipation_manage`: إنشاء الجداول الأربعة بنجاح، مع FKs وعزل الشركة.
3. على XAMPP لمعدة موجودة: أضِف وثيقة (حرجة ومنتهية → تأكّد من التحذير)، تجهيز حماية، مكوّناً، وحدث تاريخ يدوي. تأكّد من العزل بالشركة والصلاحيات ورفع الملفات.
4. **عدم الكسر:** افتح شاشات التشغيل/التايم شيت/الأسطول وتأكّد أنها تعمل كالسابق (الأبناء إضافية بحتة لا تمسّ قراءات `equipments`).

## المخرجات المطلوبة

- سكربتات SQL في `database/` للجداول الأربعة (+ مجلد التخزين المحمي للملفات).
- تعديلات `equipment_profile.php` / `get_equipment_details.php` + أي معالج/نقاط AJAX لإدارة السطور.
- ملخّص: الجداول المنشأة، آلية الإضافة/الحذف، التنبيهات، والخطافات المعلّقة (منع التشغيل عند وثيقة حرجة منتهية، والتغذية التلقائية من S08/العدّاد لاحقاً)، وتأكيد عدم الكسر.

**ابدأ بقراءة `equipment_profile.php` و`drivercontracts.php` لمطابقة النمط، ثم أنشئ الجداول والأقسام، ثم تحقّق من عدم الكسر.**
