## المهمة

**النطاق — الصفحات الثماني أعلاه فقط.**

**المشكلة المحدَّدة:** الميزة منفّذة بأربع طرق مختلفة بفئات أزرار مختلفة (`btn-group-toggle`, `toggle-group-btn`, `cg-btn`, `group-toggle`) و CSS متفرّق في ملفات/نطاقات مختلفة وآليات إخفاء مختلفة (كلاس `.group-hidden` مقابل DataTables `column().visible()`). النتيجة: سلوك إظهار/إخفاء غير موثوق على بعض الصفحات، وألوان أزرار غير متطابقة، وتضارب مفتاح `localStorage` بين صفحتي عقود السائقين والموردين (كلاهما `supplierContractGroupStates`).

## الهدف المطلوب

### 1) توحيد المنطق البرمجي (JS)
اجعل إظهار/إخفاء مجموعات الأعمدة يعمل بشكل موثوق في كل الصفحات الثماني، سواء كان الجدول كلاسيكياً (إخفاء عبر كلاس على خلايا `.group-X`) أو DataTables (عبر `column().visible()`). يجب أن:
- يتزامن كلاس `active` على الزر **دائماً** مع حالة ظهور المجموعة (عند النقر، وعند تحميل الصفحة، وعند "إظهار/إخفاء الكل").
- يعمل زر "إظهار/إخفاء الكل" على تحديث كل الأزرار والحالة ونص الزر معاً.
- في صفحات DataTables: يبقى ربط فهارس الأعمدة صحيحاً بعد إعادة الرسم/الاستجابة (responsive)، ولا تنكسر مطابقة المجموعة بالعمود.

### 2) توحيد الأزرار وألوانها (CSS) — مصدر واحد للحقيقة
وحّد كل أزرار المجموعات على **فئة واحدة** (يُفضّل `btn-group-toggle` لأنها الأوسع استخداماً) معرّفة في **ملف CSS مشترك واحد** فقط، بحيث:
- **الزر المفعَّل (active): أصفر غامق** — اعتمد التدرّج/اللون المستخدم حالياً: خلفية `linear-gradient(135deg,#D9AB32,#C2941C)`، نص `#2A2010`، حدّ `#B0851A`.
- **الزر غير المفعَّل: رمادي فاتح** — خلفية رمادية فاتحة (مثل `#F1F2F4`) مع نص/حدّ رمادي، **وليس أبيض**.
- حالة hover متّسقة.
- احذف/أزل تعريفات الفئات القديمة المكرّرة (`toggle-group-btn`, `cg-btn`, `group-toggle`) من الملفات و`<style>` الداخلية بعد الترحيل لتفادي التعارض.

### 3) إصلاح تضارب الحفظ
أعطِ كل صفحة مفتاح `localStorage` فريداً (صحّح `Drivers/drivercontracts.php` ليستخدم مثلاً `driverContractGroupStates` بدل مفتاح الموردين)، وتأكد أن حالة كل صفحة مستقلة.

## خطوات التنفيذ
1. **جرد:** افحص الصفحات الثماني وحدّد لكل صفحة: فئة الزر، آلية الإخفاء، مصدر الـ CSS، ومفتاح الحفظ (راجع المصادر المذكورة أعلاه).
2. **القاعدة الموحّدة:** ثبّت فئة الزر الموحّدة وكتلة CSS واحدة لحالاتها (active/غير مفعّل/hover) في ملف مشترك.
3. **الترحيل:** حدّث الصفحات الأربع ذات الفئات المختلفة لتستخدم الفئة وآلية الـ JS الموحّدة، مع الحفاظ على بنية كل جدول (كلاسيكي مقابل DataTables).
4. **عدم تكرار:** لا تترك CSS أو JS مكرّراً داخل الصفحات؛ المصدر الموحّد فقط (مع استثناءات محصورة عند الضرورة).

## قبل التنفيذ
اعرض خطة موجزة (الفئة الموحّدة المختارة، الملف الذي ستضع فيه الـ CSS/JS الموحّد، والتغييرات لكل صفحة). انتظر الموافقة قبل تعديل الكود.

## التحقق (إلزامي)
لكل صفحة من الثماني: اضغط كل زر مجموعة وزر "الكل"، وتأكد بصرياً (لقطات شاشة) أن:
- (أ) الأعمدة تُخفى/تظهر فعلاً.
- (ب) الزر يصبح **أصفر غامق عند التفعيل** و**رمادي فاتح عند التعطيل**.
- (ج) الحالة تُحفظ بعد إعادة التحميل.
- (د) صفحتا عقود السائقين والموردين مستقلّتان عن بعضهما.
