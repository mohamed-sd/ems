# برومت تنفيذ: نظام قطاع تشغيل المعدات (OPE) داخل مشروع EMS
> انسخ هذا النص كاملًا وسلّمه إلى Claude Code **داخل مجلد المشروع** `c:/wamp64/www/ems`، وأرفق معه ملفّي **«المرجع الشامل للمبرمج»** و**«العلاقة بين الأنظمة الثلاثة»**.
>
> ⚠️ **ملاحظة معمارية حاسمة (اقرأها أولًا):** المستودع الحالي **ليس Odoo**. هو تطبيق **PHP خام + MySQLi (إجرائي/OOP خفيف)، عربي RTL، متعدّد الشركات (Multi-tenant)**، اسمه EMS، يعمل على WAMP وقاعدة بيانات `equipation_manage`. لذلك **لن تُنشئ وحدات Odoo ولا models/views/XML ولا `ir.model.access.csv` ولا `res.groups`**. بل ستنفّذ نفس النموذج الذهني للملف المرجعي لكن **بأدوات هذا المشروع** كما هو موضّح في جدول الترجمة أدناه. التزم بـ«ماذا» في الملف المرجعي (الجداول والحقول والشاشات وآلات الحالات والصلاحيات والتكامل)، ونفّذ «كيف» باصطلاحات EMS القائمة.

## دورك
أنت مطوّر **PHP/MySQL خبير (Senior)** يعمل داخل نظام EMS قائم وناضج. ستبني **قطاع تشغيل المعدات (Equipation — OPE)** المكوَّن من ثلاث وحدات وظيفية متكاملة: **الأسطول (S03) · الصيانة (S08) · الحركة والتشغيل (S07)**، اعتمادًا حصريًّا على الملف المرجعي المرفق. التزم بأسماء الجداول والحقول وأنواعها كما وردت في الملف حرفيًّا (الأسماء البرمجية إنجليزية، الواجهة عربية)، ولا تخترع حقولًا ولا تغيّر التسميات. **أعد استخدام كل ما هو مبني مسبقًا** في المستودع بدل تكراره.

## النموذج الذهني — جدول ترجمة Odoo → EMS (إلزامي)
الملف المرجعي مكتوب بمصطلحات Odoo. هذا هو ما يقابل كلًّا منها **في هذا المشروع فعلًا**:

| المفهوم في الملف المرجعي (Odoo) | ما تنفّذه فعليًّا في EMS |
|---|---|
| الوحدة / الـ addon (الأسطول · الصيانة · الحركة) | **مجلد علوي مستقل** بنمط المشروع: `Fleet/` · `Maintenance/` · `Operations/` (أو الأسماء المعتمدة في المستودع إن وُجدت). كل شاشة = ملف `.php` |
| `res.groups` (مجموعة أمان لكل دور) | صف في جدول **`roles`** + رمز دور رقمي. الأدوار القائمة: `-1` مدير عام، `1` مدير مشاريع، `2` مدير موردين، `3` مدير مشغّلين، `4` **مدير الأسطول**، `5` مدير موقع، `6` إدخال ساعات، `7/8` مراجعة، `9` مراجعة الأعطال. أنشئ ما ينقص عبر `/admin/permissions/roles.php` |
| الشاشة / الـ view (form/tree/kanban/search) | **صفحة PHP** تعرض: نموذج Bootstrap 5 RTL + قائمة **DataTables** عربية. النموذج المنبثق (`target="new"`) = **Bootstrap Modal** |
| `ir.actions.act_window` + `menuitem` | صف في جدول **`modules`** (`code`, `name`, `icon`, `owner_role_id`, `is_link`) عبر `/admin/permissions/modules.php` + رابط في الملاح (`sidebar.php` / `includes/dynamic_nav.php`) مُقيَّد بالدور |
| `ir.model.access.csv` (CRUD لكل group) | صفوف في جدول **`role_permissions`** (`role_id`, `module_id`, `can_view`, `can_add`, `can_edit`, `can_delete`) عبر `/admin/permissions/role_permissions.php` |
| `ir.rule` (قواعد سجلات / عزل) | **عزل متعدّد الشركات**: `company_id` في كل `WHERE`، إضافةً لنطاق المشروع/الدور. استخدم `db_table_has_column()` للمرونة |
| `models` (أصناف Python + حقول) | **جداول MySQL** في `equipation_manage` + سكربتات هجرة `.sql` تحت `database/`. الوصول عبر `mysqli` (Prepared Statements) أو طبقة `App\` للخدمات |
| الحقل `state` (Selection) + `statusbar` | عمود `state`/`status` (ENUM أو VARCHAR/TINYINT) + **شريط حالة (stepper/badges) بأسلوب Bootstrap**، وأزرار انتقال تُنفَّذ عبر **JSON action handler** (`*_handler.php`) |
| حقل `related`/`computed readonly` | بيان من وحدة أخرى يُعرَض **للقراءة فقط** عبر `JOIN`/Sub-query — لا حقل إدخال. مصدر حقيقة واحد |
| `ir.sequence` (توليد أكواد) | **دالة مساعدة لتوليد الكود** (مثل `equipment_code`) بنمط `prefix-YYYY-####`، مع فحص التفرّد قبل الإدراج |
| `target="new"` (نافذة منبثقة) | **Bootstrap Modal** يُفتح بـ JS، ويُرسِل عبر AJAX إلى handler |

**الأدوار الثلاثة كلها مطلوبة.** إن وجدت أيًّا منها مبنيًّا (لاحظ: يوجد بالفعل دور «مدير الأسطول=4» وشاشة أعطال ووحدات `movement/`, `Oprators/`, `Timesheet/`, `Equipments/`)، فتحقّق وأعد الاستخدام وأكمل الناقص؛ وإلا فأنشئه من الصفر. نفّذ **كل ما في الملف**: كل قسم وكل شاشاته وتخطيطها وجداولها وحقولها وأنواعها.

## بنية كل وحدة (بأسلوب EMS — لا توجد addons)
لكل وحدة مجلد علوي مستقل، وكل شاشة ملف PHP. النمط القياسي:

```
Fleet/  (وكذلك Maintenance/  و Operations/)
├── <screen>.php                # الشاشة: حماية جلسة + صلاحيات + معالجة POST + نموذج + DataTables
├── <screen>_handler.php        # معالج JSON للأفعال (انتقالات الحالة/الاعتماد/الإغلاق/التكامل)
├── get_<something>.php          # نقاط AJAX للقوائم المنسدلة المعتمِدة (تُعيد <option> أو JSON)
└── .htaccess / index.php       # حارس وصول كما في بقية المجلدات
```
ملفات مشتركة تُضاف عند اللزوم:
- **منطق الأعمال المعقّد** (آلات الحالات، أحداث التكامل عبر الوحدات): ضعه في خدمات تحت `app/Services/Ope/` بمساحة الأسماء `App\Services\Ope` وتُحمَّل عبر `app/bootstrap.php` — حتى لا يتكرّر المنطق بين الشاشات (مصدر حقيقة واحد للمنطق).
- **سكربتات الهجرة**: `database/ope_<module>_<n>.sql` (إنشاء جداول/أعمدة + `roles`/`modules`/`role_permissions` + كتالوجات مرجعية).

**التبعية المنطقية:** الصيانة والتشغيل يقرآن هوية المعدة من الأسطول (مصدر الهوية). لا توجد آلية `depends`؛ التبعية تُحقَّق عبر `JOIN`/مفاتيح أجنبية واستدعاء خدمات `App\Services\Ope`.

## لكل شاشة، نفّذ هذه العناصر (دون نقصان)
- **الجدول (table)**: بكل حقوله كما في جدول الشاشة بالملف — الاسم الإنجليزي للحقل والنوع المقابل في MySQL (`VARCHAR`/`Char`، `ENUM`/`Selection`، `INT + FK`/`Many2one`، جدول أبناء/`One2many`، `DECIMAL`/`Float`/`Monetary`، `DATE/DATETIME`، `TINYINT(1)`/`Boolean`)، والإلزامية (`NOT NULL`)، والمصدر/القيم. اكتبه كسكربت ميجريشن `.sql` إنشاء/تعديل **آمن** (`CREATE TABLE IF NOT EXISTS` / فحص العمود قبل `ALTER`).
- **الحقول المحسوبة والمجلوبة**: ما وُسم «(محسوب)» احسبه في PHP/SQL عند العرض أو خزّنه ودوّن متى يُحدَّث؛ وما وُسم «related/قراءة» اجلبه عبر `JOIN`/Sub-query واعرضه **للقراءة فقط** (لا حقل إدخال، لا تخزين مزدوج).
- **آلة الحالات**: عمود `state` بالقيم المذكورة + **شريط حالة Bootstrap (stepper/badges)** في أعلى النموذج، وأزرار انتقال يُنفِّذها `*_handler.php` بمنطقها وشروطها، وكل زر **مُقيَّد بالصلاحية/الدور المخوَّل** (يظهر/يُخفى عبر `can_show_button` ويُتحقَّق في الخادم عبر `enforce_module_permission_json`).
- **العرض (الواجهة)**: نموذج بتخطيط الملف المقترح حرفيًّا (شريط الحالة وأزراره أعلى، مجموعتا حقول يمين/يسار، تبويبات لسطور الأبناء One2many)، + جدول **DataTables** (بحث/فرز/تصدير، لغة عربية عبر `language.url`), + مرشّحات/تجميعات مناسبة. الواجهة **RTL** والتسميات عربية.
- **الإجراء والقائمة**: سجّل الشاشة في جدول `modules` (code = المسار النسبي، `owner_role_id` = دور الوحدة، أيقونة) + رابط في الملاح مُقيَّد بالدور. النوافذ المنبثقة = Bootstrap Modal.
- **سطور الصلاحيات**: أضف صفوف `role_permissions` لهذا الـ module لكل دور معنيّ (view/add/edit/delete) ضمن سكربت الـ `.sql`.

## الأدوار والصلاحيات (التزم بها بدقّة — عبر الجداول الديناميكية)
- لكل وحدة **دوران على الأقل**: مستخدم ومدير — أنشئها في جدول `roles` (وأضف الأدوار الفرعية التي يذكرها الملف: مشرف صيانة، مسؤول حركة، مشرف وردية، مدير موقع…). أعد استخدام دور **مدير الأسطول (4)** القائم.
- **كل دور يرى ويحرّر شاشاته فقط**: التقييد عبر `role_permissions` + `owner_role_id` في `modules` + إخفاء روابط الملاح غير المسموحة. ضع في **أعلى كل صفحة** فحصًا: `check_view_permission($conn, $module_id)` (أو `get_page_permissions`)، وفي كل handler: `enforce_module_permission_json(...)`.
- **الاعتماد والإغلاق للمخوَّل حصرًا**: أزرار الاعتماد/الإقفال تظهر فقط عبر `can_show_button(...,'edit'/'add')` ويُتحقَّق منها في الخادم. راجع «مصفوفة اعتماد الصيانات مرتفعة التكلفة» وبوّابة اعتماد الجاهزية وإقفال L5.
- **الحقول العابرة للوحدات** تظهر للدور الآخر **للقراءة فقط** (related)، لا للتحرير.
- **العزل متعدّد الشركات** عبر `company_id` في كل استعلام (ابنِه على نمط العزل القائم في المشروع). ⚠️ تجنّب الـ Fallback المتساهل الذي يمنح كل الصلاحيات عند غياب تعريف الـ module (ثغرة مرصودة في تقييم المشروع) — للشاشات الجديدة اجعل الغياب = **منع**، لا سماح.

## مبدأ مصدر الحقيقة الواحد (لا تخالفه)
لكل بيانٍ مالكٌ واحدٌ يكتبه، والبقية يقرؤونه (related/computed readonly):
- **الأسطول** يملك هوية المعدة وحالتها وعدّادها وإتاحتها.
- **الصيانة** تملك الأعطال والتفتيش والتكلفة والمؤشّرات والجاهزية الفنية.
- **التشغيل** يملك القرار اليومي وسجلَّه (L3 · L4 · L5) وقرارات التوقف.

أي حقلٍ مصدره وحدة أخرى = يُجلب بـ `JOIN` ويُعرَض readonly، ولا يُخزَّن/يُدخَل مرتين. راجع **مصفوفة ملكية البيانات** في ملف العلاقات والتزم بها حرفيًّا. ضع منطق الكتابة المملوكة داخل خدمات `App\Services\Ope` لتكون نقطة الحقيقة الوحيدة.

## نقاط التكامل الحرجة التسع (نفّذها كأفعال/أحداث عابرة للوحدات في `App\Services\Ope` تُستدعى من المعالجات)
1. **اعتماد استمارة الإدخال (S03)** ← يُنشئ كرت المعدة بكودٍ دائم (دالة توليد الكود) ويفتح **تفتيش استلام** في الصيانة (S08).
2. **اعتماد الجاهزية الفنية (S08)** ← يسمح للأسطول بإتاحة المعدة في **مجمّع المتاح** (S03).
3. **التخصيص L3 / التوزيع L4 (S07)** ← تتحوّل المعدة إلى «مخصَّصة» وتخرج من المتاح (S03).
4. **بلاغ الميدان (S07)** ← بلاغ في الصيانة بسياقه ثم أمر صيانة، فتصير المعدة «قيد الصيانة» (S03).
5. **إغلاق أمر الصيانة (S08)** ← عودة المعدة «متاح»، تحديث العدّاد و«آخر صيانة»، إعادة جدولة الوقائية، تغذية المؤشّرات (S03).
6. **إقفال السجل اليومي L5 (S07)** ← **التغذية الرباعية**: `engine_hours`→الأسطول، ساعات الأجر→القوى، كميات الفوترة→الإيرادات، الفعليات→لوحة الأداء.
7. **أمر تحرك معدة (S07)** ← يفتح آليًّا ترحيل أسطول (S03) وأمر نقل (S06) إن وُجد.
8. **تفتيش ما قبل التخلص (S08)** ← شرطٌ لاستمارة الخروج (S03).
9. **تجاوز الصيانة الحرجة (S08 يحسب الاستحقاق)** ← الأسطول (S03) يحظر/يحذّر التخصيص (L3 في S07). إن وُجد **محرّك الحصص** `app/Services/Allocation/`، فادمج هذا الحظر ضمنه ولا تُكرّر منطق الفرض.

كل فعل: تحقّق من الصلاحية، نفّذ داخل **Transaction** واحدة، أعِد JSON `{success, message}` (عربي)، ولا تُكرّر الكتابة في أكثر من مكان.

## أسلوب العمل ومعايير الجودة (اصطلاحات EMS الإلزامية)
- **PHP + MySQLi + Bootstrap 5 RTL + jQuery + DataTables**؛ لا أُطر إضافية. الأسماء البرمجية إنجليزية مطابقة للملف، والواجهة عربية.
- **الأمان أولًا** (التقييم الحالي 58/100 ويجب رفعه، لا خفضه): استخدم **Prepared Statements** لكل المدخلات في الكود الجديد (لا concatenation)، و`e()` لكل المخرجات، وتوكن CSRF في كل فورم/AJAX حسّاس، و`intval()/floatval()` للأرقام، وحماية الوصول المباشر (`.htaccess`/`index.php`) لكل مجلد جديد، وحارس AJAX القائم في `config.php`.
- **نمط الصفحة**: `session check` → `include '../config.php'` → فحص صلاحية → معالجة POST (تحقّق→تنفيذ→`header()` redirect) → header template (`inheader.php`/`insidebar.php` مع `$page_title`) → نموذج + DataTables.
- **نمط المعالج (`*_handler.php`)**: تحقّق `REQUEST_METHOD==='POST'` → `enforce_module_permission_json` → توجيه بالأفعال (`$_POST['action']`) → دوال مساعدة مُغلَّفة بـ `if(!function_exists())` → `json_encode(['success'=>..,'message'=>..])` + `exit;`.
- اجعل منطق الأعمال في خدمات `App\Services\Ope` (قابلة لإعادة الاستخدام والاختبار)، والصفحات نحيفة.
- علّق المنطق غير البديهي (المحسوبات، الانتقالات، أحداث التكامل) بالعربية أو الإنجليزية.
- **بعد كل وحدة**: شغّل سكربتات الهجرة على قاعدة محلية، وتحقّق `php -l` لكل ملف، وجرّب الشاشات يدويًّا (إضافة ضمن الصلاحية تنجح، وخارجها تُمنع)، قبل الانتقال.
- لا تفترض عند الغموض — اسألني.
