# نظام الاعتماد - التغييرات المنفذة

## ملخص التنفيذ

تم تطبيق نظام الاعتماد في مكانين فقط كما طُلب:

### 1️⃣ تشغيل وإيقاف الآلية (Oprators/oprators.php)
- مدير الحركة والتشغيل (role 10) يُقدّم طلب إيقاف آلية
- مدير الأسطول (role 4) يعتمد أو يرفض

### 2️⃣ تشغيل وإيقاف المشغل (Equipments/equipments.php)
- مدير الحركة والتشغيل (role 10) يُقدّم طلب تشغيل/إيقاف مشغل
- مدير المشغلين (role 3) يعتمد أو يرفض

---

## الملفات المعدلة

| الملف | التعديل |
|------|---------|
| `insidebar.php` | إضافة رابط "طلبات الموافقات" لـ role 3 |
| `Equipments/delete_equipment_driver.php` | نظام اعتماد لإيقاف/تشغيل المشغل |
| `Equipments/save_equipment_drivers.php` | نظام اعتماد لإضافة مشغل جديد |
| `includes/approval_workflow.php` | إضافة قواعد activate_driver |
| `database/approval_workflow_driver_rules.sql` | قواعد SQL |
| `database/update_approval_rules.php` | سكريبت تحديث القواعد |

---

## قواعد الموافقة المضافة

```
✓ driver:activate_driver → role 3,-1
✓ driver:deactivate_driver → role 3,-1
✓ driver:reactivate_driver → role 3,-1
✓ equipment:deactivate_equipment → role 4,-1
✓ equipment:reactivate_equipment → role 4,-1
```

---

## كيفية الاستخدام

### مدير الحركة والتشغيل (role 10)
1. **إيقاف آلية**: 
   - التشغيل → طلب إيقاف → كتابة السبب → إرسال
   
2. **إضافة مشغل**:
   - الآليات → إدارة المشغلين → اختيار مشغل → حفظ

3. **إيقاف مشغل**:
   - الآليات → إدارة المشغلين → تبديل الحالة

### مدير الأسطول (role 4)
- طلبات الموافقات → عرض طلبات الآليات → اعتماد/رفض

### مدير المشغلين (role 3)
- طلبات الموافقات → عرض طلبات المشغلين → اعتماد/رفض

---

## النتائج

### عند الاعتماد ✅
- يتم تنفيذ التغيير تلقائيًا (تحديث status في قاعدة البيانات)
- تحديث حالة الطلب إلى "معتمدة"

### عند الرفض ❌
- **لا يتم** تنفيذ أي تغيير
- تحديث حالة الطلب إلى "مرفوضة"
- تسجيل سبب الرفض

## طريقة التشغيل

### 1. تحديث قواعد البيانات
```bash
php database/update_approval_rules.php
```

### 2. التأكد من القواعد
```sql
SELECT * FROM approval_workflow_rules 
WHERE entity_type IN ('driver', 'equipment');
```

النتيجة المتوقعة: 5 قواعد
