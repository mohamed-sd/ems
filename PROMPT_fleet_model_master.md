# برومبت: بناء «سجل النوع والموديل» + ربط المعدات بالموديل (Claude Code)

> انسخ كل ما تحت الخط والصقه في Claude Code داخل مجلد المشروع `c:/xampp/htdocs/ems`.
> هذه **المرحلة 0 — الخطوة 1** من مشروع «الأسطول والجاهزية» (راجع `ROADMAP_fleet_readiness_on_php.md`). نطاق هذا البرومبت محدود ومقصود.

---

## السياق

نظام EMS (PHP خام + MySQLi، عربي RTL، متعدّد الشركات، XAMPP). نبني وحدة «الأسطول والجاهزية» على مراحل. مبدأ التصميم: **لا يبدأ تعريف المعدة من كرتها، بل من سجل النوع والموديل** الذي يُسجَّل مرة واحدة بمواصفاته القياسية، ثم يرث الكرت منه تلقائياً (كرت أصغر، لا تكرار، مواصفات موحّدة).

هذا البرومبت ينفّذ شيئين فقط:
1. **سجل النوع والموديل** (`fleet_model`) + سطور **مواصفات الخدمة القياسية** (`fleet_model_service_spec`) — جدول + شاشة CRUD كاملة.
2. **ربط جدول `equipments` بالموديل** (`model_id`) مع وراثة بيانات الموديل في نموذج المعدة — كخطوة أولى في تحويل `equipments` إلى «كرت المعدة».

> لا تبنِ دورة الحياة ولا الإهلاك ولا الكرت الكامل في هذا البرومبت — مراحل لاحقة.

## اقرأ أولاً (لمطابقة أعراف النظام بدقّة)

- `Equipments/equipments_types.php` و`Equipments/equipments.php` — نمط شاشة CRUD المرجعية، عزل الشركة، الصلاحيات، إطار Excel، DataTables، والترويسة الموحّدة.
- `includes/permissions_helper.php` (`check_page_permissions`, `db_table_has_column`) · `config.php` · `includes/page_header.php` · `inheader.php`/`insidebar.php`.
- `app/Services/Excel/ExcelRegistry.php` و`includes/excel_ui.php` — لإضافة كيان Excel للموديلات (تصدير/استيراد) إن أمكن بسهولة.
- `database/equipation_manage.sql` — مخطّط `equipments` و`equipments_types` و`suppliers`.
- `assets/css/design-tokens.css` — رموز الهوية (أسود/كهرماني) للالتزام بها.

## 1) جدول `fleet_model` (سجل النوع والموديل)

أنشئ سكربت SQL في `database/` (مثل `fleet_model.sql`) ينشئ الجدول، مع عزل الشركة وحذف ناعم على نمط النظام:

| الحقل | النوع | إلزامي | ملاحظات |
|---|---|---|---|
| `id` | INT PK AI | — | |
| `company_id` | INT NULL | — | عزل الشركة |
| `code` | VARCHAR(50) | نعم | كود الموديل — فريد ضمن الشركة (تسلسل تلقائي أو يدوي) |
| `manufacturer` | VARCHAR(150) | نعم | الشركة المصنعة (هيونداي·كوماتسو·كاتربيلر·هيدروميك·فولفو·...) |
| `model_name` | VARCHAR(150) | نعم | مثال: R220LC |
| `equipment_type_id` | INT | نعم | **مفتاح أجنبي إلى `equipments_types.id`** (انظر ملاحظة التوحيد أدناه) |
| `operating_category` | VARCHAR(50) | — | قائمة: تحميل·نقل·تخريم·تمهيد·توليد·مساندة |
| `fuel_type` | VARCHAR(30) | — | قائمة: ديزل·بنزين·كهرباء |
| `std_capacity` | DECIMAL(10,2) NULL | — | القدرة/السعة القياسية |
| `std_capacity_uom` | VARCHAR(20) NULL | — | وحدة السعة (م³·طن·ك.و...) |
| `tech_reference` | TEXT NULL | — | بيانات فنية مرجعية من كتالوج الصانع |
| `default_supplier_id` | INT NULL | — | FK إلى `suppliers.id` — المورّد المرجعي |
| `status` | ENUM('active','inactive') DEFAULT 'active' | — | الحالة |
| `is_deleted` TINYINT, `created_by`, `created_at`, `updated_at` | — | على نمط النظام |

الحقول **المحسوبة** تُعرض ولا تُخزَّن (تُحسب عند القراءة):
- `unit_count` = عدد المعدات في `equipments` المرتبطة بهذا الموديل (`model_id`).
- `perf_kpis` (ساعات قبل العطل · كلفة الصيانة/ساعة · الجاهزية · توصية التكرار) — **اعرضها الآن كحقول فارغة/«لاحقاً»** لأن مصادرها (S08/الإهلاك) غير جاهزة؛ صمّم مكانها في الواجهة فقط.

### ملاحظة توحيد «نوع المعدة» (مهمة)
المواصفة تذكر أنواعاً أوسع (حفّار·قلّاب·لودر·قريدر·دوزر·مولّد·خرّامة·شاحنة·رافعة·أخرى)، بينما النظام يخزّن النوع في جدول `equipments_types`. **وحّدها:** اربط `fleet_model.equipment_type_id` بـ`equipments_types.id`، وأضِف الأنواع الناقصة إلى `equipments_types` (insert إن لم توجد) بدل قيم ثابتة مبعثرة. لا تُكرّر تعداداً ثابتاً في الكود.

## 2) جدول `fleet_model_service_spec` (مواصفات الخدمة القياسية §9.1-أ)

سطور تابعة للموديل (One2many):

| الحقل | النوع | إلزامي | ملاحظات |

| `company_id`, `created_at` | — | |

## 3) شاشة سجل النوع والموديل (`Equipments/fleet_models.php`)

شاشة CRUD كاملة على **نفس نمط** `equipments_types.php`/`equipments.php`:
- حارس الجلسة → `config.php` → `permissions_helper` → فحص `check_page_permissions` (view/add/edit/delete) → عزل الشركة (`company_id`) عبر `db_table_has_column`.
- ترويسة موحّدة عبر `includes/page_header.php`، RTL، رموز الهوية، DataTables بالعربية مع أزرار التصدير.
- **نموذج إضافة/تعديل** يضم كل حقول `fleet_model` أعلاه، **مع جدول سطور قابل للتكرار** لمواصفات الخدمة (إضافة/حذف سطر + رفع صورة لكل سطر) — على نمط جداول السطور الموجودة في النظام (مثل سطور معدات العقد في `drivercontracts.php`).
- **قوائم منسدلة:** المصنّع (حر أو قائمة)، نوع المعدة من `equipments_types`، الفئة التشغيلية/الوقود/الوحدة (قوائم ثابتة)، المورّد المرجعي من `suppliers`.
- جدول العرض: الكود، المصنّع، الموديل، النوع، الفئة، عدد الوحدات (محسوب)، الحالة، وإجراءات (تعديل/تعطيل/عرض). **الحذف تعطيل ناعم** (status=inactive) لا حذف فعلي (كما في `equipments_types`).
- كل المخرجات عبر `htmlspecialchars`/`e()`، وكل الاستعلامات **Prepared Statements**، وكل عمليات الكتابة بـ CSRF.
- (إن سهُل) أضِف كيان `fleet_models` لإطار Excel الموحّد (`ExcelRegistry` + `ems_excel_header_actions`) للتصدير/الاستيراد.

## 4) ربط `equipments` بالموديل (أول خطوة نحو الكرت)

- أضِف عمود `model_id INT NULL` إلى `equipments` (FK إلى `fleet_model.id`)، بشكل **غير كاسر** (ALTER آمن + `db_table_has_column`).
- في نموذج المعدة (`equipments.php` و`equipments_fleet.php`): أضِف قائمة اختيار **الموديل**، وعند اختياره **يجلب تلقائياً** (للقراءة/التعبئة المبدئية) النوع والمصنّع والفئة من `fleet_model` (وراثة الموديل). أبقِ الحقول قابلة للتعديل لكن مُعبّأة من الموديل.
- لا تكسر أي منطق قائم: شاشات التشغيل/التايم شيت تقرأ من `equipments` كما هي؛ التغيير **إضافي فقط**.
- أضِف الموديل (إن وُجد) إلى عرض تفاصيل المعدة.

## 5) الصلاحيات والتنقّل

- أضِف وحدة الشاشة إلى جدول `modules` (تابعة للدور 3 «إدارة الأسطول») بكود `Equipments/fleet_models.php` وأيقونة مناسبة، حتى تظهر في القائمة الديناميكية وتُفحص صلاحياتها.
- رابط رجوع للوحة التحكم، ورابط من شاشة الأسطول إلى سجل الموديلات.

## التحقق (إلزامي قبل الإنهاء)

1. `php -l` على كل ملف معدّل/جديد — بلا أخطاء.
2. شغّل سكربتات SQL على قاعدة `equipation_manage` وتأكّد من إنشاء الجدولين والعمود الجديد دون كسر.
3. على XAMPP: أنشئ موديلاً مع عدّة سطور مواصفات خدمة (مع صورة) → عدّله → عطّله → تأكّد من العزل بالشركة والصلاحيات.
4. افتح نموذج المعدة، اختر موديلاً، وتأكّد من **وراثة** النوع/المصنّع/الفئة، ومن أن إضافة/تعديل معدة قائمة ما زالت تعمل وأن شاشات التشغيل/التايم شيت لم تتأثّر.

## المخرجات المطلوبة

- سكربتات SQL في `database/` (`fleet_model`, `fleet_model_service_spec`, إضافة `model_id` لـ`equipments`, إدراج الوحدة في `modules`، وأي أنواع ناقصة في `equipments_types`).
- `Equipments/fleet_models.php` (الشاشة) + أي معالج/نقاط AJAX لازمة.
- تعديلات `equipments.php`/`equipments_fleet.php` لربط الموديل ووراثته.
- ملخّص: الملفات المنشأة/المعدّلة، وتأكيد عدم كسر النظام، وأي افتراض اتخذته.

**ابدأ بقراءة `equipments_types.php` و`equipments.php` لمطابقة النمط، ثم أنشئ الجداول والشاشة، ثم اربط المعدة بالموديل، ثم تحقّق.**
