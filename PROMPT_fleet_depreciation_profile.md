# برومبت: بناء «ملف الافتراضات المالية والإهلاك» (§9.2) — Claude Code

> انسخ كل ما تحت الخط والصقه في Claude Code داخل مجلد المشروع `c:/xampp/htdocs/ems`.
> **المرحلة 0 — الخطوة 2** من مشروع «الأسطول والجاهزية» (راجع `ROADMAP_fleet_readiness_on_php.md`). يأتي بعد «سجل النوع والموديل» (`PROMPT_fleet_model_master.md`). نطاق محدود ومقصود.

---

## السياق

نظام EMS (PHP خام + MySQLi، عربي RTL، متعدّد الشركات، XAMPP). نبني وحدة «الأسطول والجاهزية» على مراحل.

**المبدأ:** لا تُدخَل افتراضات الإهلاك على كرت المعدة مباشرةً؛ بل تُعرَّف **مرة واحدة** في ملف افتراضات لكل **فئة أصل** (وقد تُخصَّص لماركة أو موديل)، وتُجلب تلقائياً عند إنشاء الكرت. **مالك الملف ومُعتمِده الإدارة المالية**، والأسطول **يقرأ منه ويحسب عليه** ولا يعدّل قيمه.

هذا البرومبت ينفّذ **ملف الافتراضات المالية والإهلاك فقط** (الجدول + الشاشة + قواعد الاعتماد + البذور الافتراضية). **لا** تبنِ حساب الإهلاك الفعلي على الكرت ولا تقرير الإهلاك الشهري الآن (مرحلة لاحقة) — هذا الملف **مرجع** يُقرأ منه.

## اقرأ أولاً (لمطابقة أعراف النظام)

- `Equipments/equipments_types.php` — نمط شاشة CRUD مرجعية بسيطة (إضافة/تعديل/تعطيل، عزل الشركة، الصلاحيات، Excel، DataTables).
- `Equipments/fleet_models.php` — **سجل النوع والموديل** الذي بُني في الخطوة السابقة (لربط `model_id`). إن لم يكن موجوداً بعد، اجعل ربط الموديل اختيارياً ومتسامحاً عبر `db_table_has_column`/وجود الجدول.
- `includes/permissions_helper.php` (`check_page_permissions`, `db_table_has_column`) · `config.php` · `includes/page_header.php` · `assets/css/design-tokens.css`.

## 1) جدول `fleet_depreciation_profile`

أنشئ سكربت SQL في `database/` (مثل `fleet_depreciation_profile.sql`) على نمط النظام (عزل شركة + حذف ناعم + created/updated):

| الحقل | النوع | إلزامي | ملاحظات |
|---|---|---|---|
| `id` | INT PK AI | — | |
| `company_id` | INT NULL | — | عزل الشركة |
| `code` | VARCHAR(50) | نعم | كود الملف — فريد ضمن الشركة (تسلسل تلقائي مقترح) |
| `asset_category` | VARCHAR(120) | نعم | فئة الأصل (مثل: «حفّار 22ط جديد»، «قلّاب مستعمل»...) |
| `brand` | VARCHAR(120) NULL | — | اختياري لتخصيص الافتراض لماركة |
| `model_id` | INT NULL | — | FK إلى `fleet_model.id` (اختياري — يربط بسجل §9.1) |
| `method` | ENUM('uop','sl') | نعم | `uop` = بالساعة التشغيلية · `sl` = زمني بالسنوات |
| `useful_life` | DECIMAL(12,2) | نعم | بالساعات إن `uop`، وبالسنوات إن `sl` |
| `salvage_pct` | DECIMAL(5,4) | نعم | نسبة القيمة التخريدية من تكلفة الشراء (مثل 0.08) |
| `notes` | TEXT NULL | — | افتراضات/سياسات مالية إضافية |
| `state` | ENUM('draft','approved') DEFAULT 'draft' | نعم | مسودة · معتمد |
| `approved_by` | INT NULL | — | FK إلى `users.id` (المدير المالي) |
| `approved_at` | DATETIME NULL | — | يُملأ عند الاعتماد |
| `is_deleted` TINYINT, `created_by`, `created_at`, `updated_at` | — | على نمط النظام |

> **تحقّق منطقي:** `salvage_pct` بين 0 و1؛ `useful_life > 0`. وحدة `useful_life` تُعرض ديناميكياً (ساعة/سنة) حسب `method`.

## 3) شاشة الملف (`Equipments/fleet_depreciation_profiles.php`)

شاشة CRUD كاملة على نمط `equipments_types.php`:
- حارس الجلسة → `config.php` → الصلاحيات (`check_page_permissions`) → عزل الشركة.
- ترويسة موحّدة (`page_header.php`)، RTL، رموز الهوية، DataTables عربية + أزرار التصدير، وإطار Excel الموحّد إن سهُل.
- **نموذج إضافة/تعديل** بكل حقول الجدول. وحدة `useful_life` تتبدّل (ساعة/سنة) حسب `method` عبر JS بسيط.
- **زر «اعتماد»** يحوّل `state` من `draft` إلى `approved` ويملأ `approved_by`/`approved_at` — متاح لمن يملك صلاحية الاعتماد (راجع قواعد الأعمال).
- جدول العرض: الكود، فئة الأصل، الماركة/الموديل، الطريقة، العمر، نسبة التخريد، الحالة (مسودة/معتمد) بشارة لونية، والإجراءات (تعديل/اعتماد/تعطيل). **الحذف تعطيل ناعم.**
- كل المخرجات `htmlspecialchars`/`e()`، كل الاستعلامات **Prepared Statements**، الكتابة بـ CSRF.

## 4) قواعد الأعمال الإلزامية (من §9.2)

1. **الملف ملك الإدارة المالية واعتماده للمدير المالي؛ والكرت يربط به ولا يعدّل قيمه.** أي أن كرت المعدة لاحقاً سيحمل `depreciation_profile_id` (FK) ويقرأ القيم منه فقط — لا نسخ ولا تعديل على الكرت.
2. **تغيير الافتراض لا يُعدِّل إهلاك الشهور المُرحَّلة؛ يسري على الحساب اللاحق فقط** (بأثر مستقبلي). وثّق هذا في الكود/الواجهة، وعند التعديل سجّل أثراً تدقيقياً (من غيّر، متى) — لا تحذف القيمة القديمة بصمت.
3. الأعمار والنسب تُراجَع دورياً (IAS 16 §51)؛ التعديل يسري من شهر التعديل فصاعداً.

> لا تُنفّذ حساب الإهلاك هنا — فقط خزّن الافتراضات. الحساب (معدل الساعة = (التكلفة − التخريد) ÷ العمر؛ المتراكم = engine_hours × المعدل) يأتي في مرحلة الكرت/الإهلاك لاحقاً، وسيقرأ من هذا الملف.

## 5) الربط على مستوى **الموديل** لا المعدة (مهم)

> الربط يكون بـ**سجل النوع والموديل** (`fleet_model`)، **وليس** بجدول المعدات مباشرة. المعدة (الكرت) ترث ملف الافتراضات من **موديلها** تلقائياً.

- أضِف عمود `depreciation_profile_id INT NULL` إلى **`fleet_model`** (FK إلى `fleet_depreciation_profile.id`) بشكل **إضافي آمن** (`db_table_has_column` + ALTER آمن). **لا تضِف هذا العمود إلى `equipments`.**
- في نموذج **سجل النوع والموديل** (`Equipments/fleet_models.php`): أضِف قائمة اختيار **ملف الافتراضات المالية** تعرض الملفات المعتمدة (`state=approved`) ضمن الشركة فقط.
- المعدة ترث الملف عبر موديلها: عند الحاجة لاحقاً (الكرت/الإهلاك) يُقرأ الملف بالمسار **المعدة ← موديلها (`equipments.model_id`) ← `fleet_model.depreciation_profile_id` ← `fleet_depreciation_profile`**. لا تربط الملف على المعدة فرادى.
- لا تحسب إهلاكاً بعد — مجرد ربط مرجعي على الموديل. ولا تكسر أي شاشة قائمة؛ التغيير إضافي فقط.

> ملاحظة: يبقى حقلا `brand` و`model_id` داخل **ملف الافتراضات** نفسه (من القاموس أعلاه) كبيانات تخصيص/تصفية اختيارية للملف، لكن **مسار الوراثة المعتمد** هو موديل → ملف → كرت كما أعلاه.

## 6) الصلاحيات والتنقّل

- أضِف وحدة الشاشة إلى `modules` (يفضّل تحت دور إداري/مالي مناسب — وضّح اختيارك) بكود `Equipments/fleet_depreciation_profiles.php` وأيقونة مناسبة، لتُفحص صلاحياتها وتظهر في القائمة.
- رابط رجوع للوحة التحكم، ورابط من شاشة الأسطول/الموديلات إليها.

## التحقق (إلزامي قبل الإنهاء)

1. `php -l` على كل ملف معدّل/جديد — بلا أخطاء.
2. شغّل SQL على `equipation_manage`: تأكّد من إنشاء الجدول، إدراج البذور الافتراضية، وإضافة العمود لـ`equipments` دون كسر.
3. على XAMPP: أنشئ ملف افتراضات (مسودة) → اعتمده → عدّله وتأكّد من سريان الأثر المستقبلي والتدقيق → عطّله → تأكّد من العزل بالشركة والصلاحيات وتبدّل وحدة العمر حسب الطريقة.
4. افتح نموذج **سجل النوع والموديل** وتأكّد من ظهور قائمة الملفات المعتمدة وإمكانية ربط الموديل بملف، وأن إضافة/تعديل موديل قائم لم تتأثّر. (لا تغيير على نموذج المعدة في هذه الخطوة.)

## المخرجات المطلوبة

- سكربتات SQL في `database/` (الجدول + البذور + عمود `fleet_model.depreciation_profile_id` + إدراج الوحدة في `modules`). **لا عمود جديد على `equipments`.**
- `Equipments/fleet_depreciation_profiles.php` (الشاشة) + أي معالج لازم.
- تعديل نموذج **سجل النوع والموديل** (`fleet_models.php`) لإضافة ربط ملف الافتراضات.
- ملخّص: الملفات المنشأة/المعدّلة، تأكيد عدم الكسر، وأي افتراض اتخذته (خصوصاً مَن يملك صلاحية الاعتماد).

**ابدأ بقراءة `equipments_types.php` لمطابقة النمط، ثم أنشئ الجدول والبذور والشاشة، ثم اربط الكرت تمهيدياً، ثم تحقّق.**
