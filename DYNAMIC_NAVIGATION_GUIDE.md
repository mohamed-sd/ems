
# نظام الروابط الديناميكية - Dynamic Navigation System

## نظرة عامة
تم تحسين الشريط الجانبي (Sidebar) ليعرض الروابط الصفحات بناءً على **دور المستخدم المسجل دخول** من طاولة قاعدة البيانات `modules`.

---

## خطوات التثبيت

### 1️⃣ تشغيل ملف قاعدة البيانات
قم بتنفيذ الملف التالي في phpMyAdmin أو MySQL Client:

**المسار:** `database/create_modules_table.sql`

هذا الملف سيقوم بـ:
- ✅ إنشاء جدول `modules` (إن لم يكن موجوداً)
- ✅ إضافة بيانات تجريبية لجميع الأدوار الموجودة في النظام
- ✅ إنشاء الروابط المناسبة لكل دور

### 2️⃣ التحديث تلقائي
تم بالفعل تحديث الملف التالي:

**المسار:** `insidebar.php`
- تم إضافة استدعاء لدالة `renderDynamicNavigation()` 
- تم الحفاظ على جميع الروابط القديمة (لم يتم حذف أي شيء)
- الروابط الديناميكية تظهر في **الأعلى** ثم تتبعها الروابط الثابتة

---

## شرح الملفات الجديدة

### 📄 `database/create_modules_table.sql`
يحتوي على:
- تعريف جدول `modules` مع جميع الحقول المطلوبة
- بيانات تجريبية لكل دور:
  - المسؤول الأعلى (role_id = -1)
  - مدير المشاريع (role_id = 1)
  - مدير الموردين (role_id = 2)
  - مدير المشغلين (role_id = 3)
  - مدير الأسطول (role_id = 4)
  - مدير الموقع (role_id = 5)
  - مدخل الساعات (role_id = 6)
  - مراجع الساعات (role_id = 7, 8)
  - مراجع الأعطال (role_id = 9)
  - مدير الحركة والتشغيل (role_id = 10)

### 📁 `includes/dynamic_nav.php`
يحتوي على 3 دوال رئيسية:

```php
// 1. جلب الروابط من قاعدة البيانات
getDynamicNavLinks($conn, $roleId)

// 2. طباعة الروابط كـ HTML
printDynamicNavLinks($links, $basePrefix = '../')

// 3. الدالة المدمجة (تجمع بين 1 و 2)
renderDynamicNavigation($conn, $roleId, $basePrefix = '../')
```

## هيكل جدول `modules`

```sql
CREATE TABLE `modules` (
  `id` int(11) PRIMARY KEY AUTO_INCREMENT,
  `name` varchar(100) - اسم الصفحة بالعربية
  `code` varchar(255) - رابط الصفحة (مثل: Clients/clients.php)
  `owner_role_id` int(11) - معرف الدور المسؤول عن الصفحة
  `icon` varchar(100) - أيقونة FontAwesome
  `status` tinyint(1) - 1 = نشط، 0 = معطل
  `sort_order` int(11) - ترتيب العرض (يرتفع أولاً الأصغر)
  `created_at` timestamp - وقت الإنشاء
  `updated_at` timestamp - وقت التحديث الأخير
);
```

## الكود الذي تم إضافته في `insidebar.php`

```php
<?php 
// عرض الروابط الديناميكية من جدول modules بناءً على دور المستخدم
if (isset($_SESSION['user']) && isset($_SESSION['user']['role']) && isset($conn)) {
  renderDynamicNavigation($conn, $_SESSION['user']['role'], '../');
}
?>
```

يتم استدعاء هذا الكود **بعد رابط الرئيسية مباشرة** وقبل الروابط الثابتة.

## مميزات النظام

✅ **ديناميكي 100%** - يمكن إضافة/تعديل/حذف الروابط من قاعدة البيانات فقط
✅ **آمن** - استخدام MySQLi مع حماية من SQL injection
✅ **مرن** - يدعم أيقونات مخصصة لكل رابط
✅ **منظم** - حقل `sort_order` للتحكم في ترتيب العرض
✅ **سهل التحديث** - لا حاجة لتعديل PHP للإضافة/الحذف
✅ **يحافظ على الروابط القديمة** - لا يتم حذف أي شيء

## تفاعل مع الواقع

### مثال 1: مدير المشاريع (role_id = 1)
عند تسجيل دخول مدير المشاريع، سيرى:
- الرئيسية
- **قائمة العملاء** ← من modules
- **المشاريع** ← من modules
- **المستخدمين** ← من modules
- **تقارير العقود** ← من modules
- **أنواع المعدات** ← من modules
- (ثم باقي الروابط الثابتة من الكود القديم)

### مثال 2: مدخل الساعات (role_id = 6)
- الرئيسية
- **ساعات العمل** ← من modules
- **ساعات اليوم** ← من modules
- (ثم باقي الروابط الثابتة)

## نصائح اضافية

📝 **للتحديث النهائي للروابط الثابتة:**
بمجرد استقرار الروابط الديناميكية، يمكن إعادة تنظيم `insidebar.php` لحذف الروابط الثابتة المكررة (لكن ليس الآن للأمان).

🔧 **للأداء:**
إذا كان هناك الكثير من الروابط، يمكن إضافة caching للنتائج من قاعدة البيانات.

🌐 **للترجمة:**
جدول `modules` يدعم الأسماء بالعربية والإنجليزية عند الحاجة (إضافة حقل إضافي `name_en` إذا لزم الأمر).

## استكشاف الأخطاء

### المشكلة: الروابط لا تظهر
**الحل:**
- تأكد من تشغيل ملف SQL `create_modules_table.sql`
- تأكد من أن المستخدم مسجل دخول بشكل صحيح (`$_SESSION['user']`)
- تأكد من أن `$conn` متاح في `insidebar.php`

### المشكلة: أخطاء في قاعدة البيانات
**الحل:**
- تحقق من أن جدول `roles` موجود
- تأكد من أن role_id في جدول `modules` يطابق role_id في جدول `roles`
