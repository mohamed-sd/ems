# برومبت: تحويل `equipments` إلى «كرت المعدة» — حقول الهوية الأساسية (Claude Code)

> انسخ كل ما تحت الخط والصقه في Claude Code داخل مجلد المشروع `c:/xampp/htdocs/ems`.
> **المرحلة 0 — الخطوة 3** من مشروع «الأسطول والجاهزية» (راجع `ROADMAP_fleet_readiness_on_php.md`). يأتي بعد «سجل النوع والموديل» و«ملف الافتراضات المالية». **النطاق: حقول الهوية الأساسية فقط — بلا جداول أبناء.**

---

## السياق والقرار

نظام EMS (PHP خام + MySQLi، عربي RTL، متعدّد الشركات، XAMPP). نحوّل **جدول `equipments` الحالي ليصبح «كرت المعدة»** مباشرةً (تجاوزنا استمارة إدخال الأصل المنفصلة). الكرت يرث من الموديل وملف الإهلاك، ويُنشأ من نموذج المعدة الحالي موسّعاً.

**نطاق هذه الخطوة محدود ومقصود:**
- إضافة **حقول الهوية الأساسية الناقصة** فقط إلى `equipments`.
- ضبط **الوراثة من الموديل** (`model_id` → النوع/المصنّع/الفئة).
- إضافة **حقل حالة الكرت** + اعتماد خفيف.
- **لا** تنشئ جداول أبناء (الوثائق/الحماية/المكوّنات/التاريخ) — مرحلة لاحقة.
- **لا** تحسب إهلاكاً ولا `engine_hours` تراكمياً — مرحلة لاحقة.
- **دون كسر** شاشات التشغيل/التايم شيت/الأسطول التي تقرأ من `equipments`.

## اقرأ أولاً

- `Equipments/equipments.php` و`Equipments/equipments_fleet.php` — نموذج المعدة الحالي وأعمدته وأقسامه (مصدر الحقيقة للنمط).
- `Equipments/fleet_models.php` (سجل النوع والموديل — بُني في الخطوة 1) و`fleet_depreciation_profiles.php` (الخطوة 2) — للوراثة.
- `includes/permissions_helper.php` (`db_table_has_column`, `check_page_permissions`) · `config.php` · `database/equipation_manage.sql` (مخطّط `equipments`).
- `movement/*`, `Timesheet/*`, `Oprators/*` — للتأكد أن قراءاتها من `equipments` لن تتأثّر.

## 1) الأعمدة الجديدة على `equipments` (ALTER آمن غير كاسر)

أضِفها عبر سكربت SQL في `database/` (مثل `equipment_card_identity.sql`)، كلها **`NULL` افتراضياً** ومع فحص `db_table_has_column` قبل أي استخدام:

| العمود | النوع | ملاحظات |
|---|---|---|
| `model_id` | INT NULL | FK إلى `fleet_model.id` (إن لم يكن أُضيف في الخطوة 1، أضِفه هنا) |
| `operating_category` | VARCHAR(50) NULL | موروث من الموديل |
| `origin_country` | VARCHAR(100) NULL | بلد الصنع |
| `engine_no` | VARCHAR(100) NULL | رقم الموتور |
| `plate_no` | VARCHAR(50) NULL | رقم اللوحة |
| `capacity` | DECIMAL(12,2) NULL | السعة/القدرة/الحمولة |
| `capacity_uom` | VARCHAR(20) NULL | وحدة السعة (م³·طن·ك.و) |
| `dimensions` | VARCHAR(200) NULL | المقاسات الفنية (طول·عرض·ارتفاع·وزن) |
| `source_type` | VARCHAR(30) NULL | قائمة: ملك · مموَّل · حق استخدام · خدمة |
| `entry_date` | DATE NULL | تاريخ دخول المعدة |
| `acquisition_cost` | DECIMAL(15,2) NULL | تكلفة الشراء (منفصلة عن `estimated_value` المقدّرة) |
| `acquisition_currency` | VARCHAR(10) NULL | عملة التكلفة (USD/SDG) |
| `opening_meter` | DECIMAL(12,2) NULL | العداد الافتتاحي (أساس التراكم لاحقاً) |
| `meter_uom` | VARCHAR(20) NULL DEFAULT 'ساعات' | وحدة العدّاد (ساعات/كيلومترات) |
| `meter_source` | VARCHAR(30) NULL | مصدر العدّاد (عداد المعدة·GPS·السجل اليومي·تقديري) |
| `card_state` | VARCHAR(20) NOT NULL DEFAULT 'draft' | حالة الكرت: `draft` (مسودة) · `active` (نشط/معتمد) |
| `card_approved_by` | INT NULL | FK إلى `users.id` |
| `card_approved_at` | DATETIME NULL | يُملأ عند الاعتماد |

> **لا تُضِف** `depreciation_profile_id` على `equipments` — هو على `fleet_model` (المعدة ترث عبر موديلها). **لا تنشئ** أي جدول أبناء في هذه الخطوة.

## 2) الوراثة من الموديل (في نموذج المعدة)

في `equipments.php` و`equipments_fleet.php`:
- إن لم تكن قائمة **الموديل** مضافة بعد، أضِفها (تعرض موديلات الشركة النشطة).
- عند اختيار الموديل، **املأ تلقائياً** (للقراءة المبدئية، مع إبقائها قابلة للتعديل): نوع المعدة (`type` من `equipment_type_id` في الموديل)، المصنّع (`manufacturer`)، والفئة التشغيلية (`operating_category`). استخدم نقطة AJAX خفيفة (مثل `Equipments/get_model_data.php`) تُرجع بيانات الموديل JSON.
- وضّح بصرياً أن هذه الحقول «موروثة من الموديل» (مع إمكانية تجاوزها).

## 3) حالة الكرت والاعتماد الخفيف (حوكمة مبسّطة)

- الكرت الجديد يُنشأ بحالة `card_state = 'draft'`.
- زر **«اعتماد الكرت»** يحوّله إلى `active` ويملأ `card_approved_by`/`card_approved_at` (لمن يملك صلاحية مناسبة من دور الأسطول).
- اعرض شارة حالة الكرت (مسودة/نشط) في الجدول وبطاقة التفاصيل.
- **مبدأ غير كاسر:** لا تمنع أي منطق تشغيل قائم بناءً على `card_state` الآن (شاشات التشغيل/التايم شيت تعمل كما هي). حالة الكرت **عرضية/حوكمية** في هذه الخطوة فقط — أبقِ خطافاً (TODO معلّق) لربطها لاحقاً بقاعدة «لا تخصيص إلا لكرت نشط» عند بناء آلة الحالات.

## 4) النموذج والعرض

- أضِف الحقول الجديدة إلى نموذج إضافة/تعديل المعدة في **قسم منطقي** (مثلاً «الهوية والمصدر» و«العدّاد») دون إثقال، مع إبقاء الأقسام الحالية كما هي.
- أضِف الحقول الجديدة إلى نافذة/صفحة تفاصيل المعدة وملفها (`equipment_profile.php` و`get_equipment_details.php`) للعرض.
- قوائم ثابتة: `source_type`, `meter_uom`, `meter_source`, `capacity_uom`, `acquisition_currency`.

## 5) عدم الكسر (إلزامي)

- كل الأعمدة جديدة و`NULL` — لا تغيّر دلالة أي عمود قائم (`code`, `type`, `status`, `availability_state/status`...).
- لا تعدّل استعلامات التشغيل/التايم شيت/الأسطول التي تقرأ من `equipments` إلا إضافة عرضية.
- استخدم `db_table_has_column` قبل قراءة/كتابة أي عمود جديد (توافق رجعي).
- كل الكتابة بـ Prepared Statements + CSRF، وكل المخرجات `htmlspecialchars`/`e()`، وعزل الشركة كما هو.

## التحقق (إلزامي قبل الإنهاء)

1. `php -l` على كل ملف معدّل/جديد — بلا أخطاء.
2. شغّل SQL على `equipation_manage`: تأكّد من إضافة الأعمدة دون كسر، وأن البيانات الحالية سليمة.
3. على XAMPP: أضِف معدة جديدة، اختر موديلاً وتأكّد من **وراثة** النوع/المصنّع/الفئة، واملأ حقول الهوية الجديدة، احفظ، ثم **اعتمد الكرت** وتأكّد من تغيّر الحالة والتدقيق.
4. **اختبار عدم الكسر:** افتح شاشة التشغيل (`move_oprators.php`) والتايم شيت وإدارة الأسطول، وتأكّد أن المعدات تظهر وتعمل كالسابق تماماً، وأن إضافة/تعديل معدة قائمة لم يتأثّر.

## المخرجات المطلوبة

- سكربت SQL في `database/` (الأعمدة الجديدة + أي ربط `modules` عند اللزوم).
- تعديلات `equipments.php` / `equipments_fleet.php` / `equipment_profile.php` / `get_equipment_details.php` + نقطة `get_model_data.php` للوراثة.
- ملخّص: الأعمدة المضافة، آلية الوراثة، حالة الكرت، وتأكيد عدم كسر شاشات التشغيل/التايم شيت/الأسطول، وأي افتراض اتخذته (خصوصاً من يملك صلاحية اعتماد الكرت).

**ابدأ بقراءة `equipments.php` و`fleet_models.php` لمطابقة النمط، ثم أضِف الأعمدة والوراثة وحالة الكرت، ثم تحقّق من عدم الكسر.**
