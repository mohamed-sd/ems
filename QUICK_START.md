### الطريقة الأولى: التثبيت التلقائي (الأسهل) ⭐

1. **افتح المتصفح** وانتقل إلى:

3. **تم!** الجدول والبيانات موجودة الآن ✓

4. **احذف** الملف `setup_dynamic_nav.php` (لأسباب أمنية)

1. افتح `phpMyAdmin`
2. اختر قاعدة البيانات `equipation_manage`
3. انسخ محتوى `database/create_modules_table.sql`
4. الصق في تبويب "SQL" وشغّل ✓

## 📊 هيكل جدول modules

```
modules
├── id (معرف)
├── name (اسم الصفحة بالعربية)
├── code (رابط الصفحة مثل: Clients/clients.php)
├── owner_role_id (معرف الدور)
├── icon (أيقونة FontAwesome)
├── status (1=نشط، 0=معطل)
├── sort_order (ترتيب العرض)
├── created_at (التاريخ/الوقت)
└── updated_at (آخر تحديث)
```

---

## 🎯 كيفية العمل

```
المستخدم يسجل دخول
        ↓
يتم قراءة role_id من $_SESSION
        ↓
يتم جلب جميع الروابط من جدول modules 
التي لها owner_role_id = role_id المستخدم
        ↓
يتم عرض الروابط في الشريط الجانبي (Sidebar)
في الأعلى ثم الروابط الثابتة
```

## 💡 أمثلة على الاستخدام

### إضافة رابط جديد عبر MySQL:

```sql
INSERT INTO modules (name, code, owner_role_id, icon, status, sort_order)
VALUES ('اسم الصفحة', 'Folder/page.php', 1, 'fa fa-icon', 1, 1);
```

### إخفاء رابط (بدون حذف):

```sql
UPDATE modules SET status = 0 WHERE id = 5;
```

### تغيير ترتيب الروابط:

```sql
UPDATE modules SET sort_order = 10 WHERE id = 5;
```

## 🔧 استكشاف الأخطاء

### الروابط لا تظهر؟

**تحقق من:**
- ✅ هل جدول modules موجود؟ 
  ```sql
  SHOW TABLES LIKE 'modules';
  ```
- ✅ هل المستخدم مسجل دخول بشكل صحيح؟
- ✅ هل role_id صحيح؟
  ```sql
  SELECT * FROM modules WHERE owner_role_id = 1;
  ```

### خطأ في قاعدة البيانات؟

**حاول:**
1. احذف جدول modules القديم:
   ```sql
   DROP TABLE IF EXISTS modules;
   ```
2. شغّل `database/create_modules_table.sql` مجدداً
3. تحديث الصفحة في المتصفح
